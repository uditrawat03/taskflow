# ADR-001: Use JSON Files for Task Storage

**Date:** Day 08
**Status:** Accepted — superseded by ADR-003 (SQLite, Day 34)
**Decider:** Udit Rawat

## Context

TaskFlow AI needed a persistence mechanism for task data.
Options evaluated: plain text, JSON files, SQLite, PostgreSQL.

## Decision

Use JSON files with atomic writes (`Path.rename()` from `.tmp`).

## Rationale

- Zero dependencies — no database driver or server required
- Human-readable — tasks viewable/editable in any text editor
- Debuggable — corrupt state is visible without special tools
- Sufficient for Phase 1 — 10–100 tasks fits in one JSON file
- Clean migration path — `to_dict()` / `from_dict()` means only `json_store.py` changes when backend changes

## Rejected Alternatives

- **Plain text** — custom serialisation is fragile; pipe separator breaks on `|` in titles
- **SQLite** — overkill for Phase 1; adds schema migrations without benefit at this scale
- **PostgreSQL** — requires a running server; inappropriate for a standalone CLI

## Consequences

- Not suitable for concurrent access
- Full file rewrite on every save — degrades at thousands of tasks
- No server-side querying — all filtering done in Python
